# Setting up your game files

This is the manual setup that the patcher can't fully automate, because
the first step needs a GUI tool that runs against your own Steam install.

## 1. Re-install the game from your Steam library

If you owned DLBB before Steam delisted it, it's still in your library.
Right-click → Install. The download is ~15 GB.

## 2. Copy the install to a working folder

The patcher modifies files in place. Run it against a **copy** of your
Steam install, not the original:

```cmd
xcopy "C:\Program Files (x86)\Steam\steamapps\common\Dying Light Bad Blood" ^
      "C:\Games\DLBB-Revival" /E /I /H
```

This way Steam Verify-Integrity won't keep undoing your patches.

## 3. Unpack the Steam DRM stub on BadBloodGame.exe

The exe ships with **SteamStub Variant 3.1 (x64)**. Without removing it,
the game refuses to start outside of Steam:

- With Steam open: "Application load error V:0000065432"
- With Steam closed: opens Steam and asks to install
- With Goldberg loader: "Application load error 3:0000065432"

Tool: [Steamless v3.1.0.5](https://github.com/atom0s/Steamless/releases) by atom0s.

```cmd
:: Drop your packed BadBloodGame.exe onto Steamless.exe (or use --d the CLI)
:: It produces BadBloodGame.exe.unpacked next to the original.

cd "C:\Games\DLBB-Revival"
ren BadBloodGame.exe BadBloodGame.exe.packed
ren BadBloodGame.exe.unpacked BadBloodGame.exe
```

Expected file size change: ~1,154,680 bytes packed → ~975,272 bytes unpacked.

## 4. Run the patcher

```cmd
cd C:\path\to\BadBlood-Revival\patcher
python apply_patches.py ^
    --game-dir "C:\Games\DLBB-Revival" ^
    --server-host pls.example.it
```

See [`patcher/README.md`](../patcher/README.md) for what this does.

## 5. Set the Steam App ID

The game expects AppID `766370` (original) but Goldberg works best with the
public-test `480` (Spacewar). Edit `steam_appid.txt` in the game folder:

```
480
```

## 6. Drop in the stubs and hook

From the built binaries (or downloaded from project Releases):

```
C:\Games\DLBB-Revival\BEClient_x64.dll               ← from stubs/
C:\Games\DLBB-Revival\BEServer_x64.dll               ← from stubs/
C:\Games\DLBB-Revival\LightFX.dll                    ← from texture-hook/
C:\Games\DLBB-Revival\LightFX_original.dll           ← rename the real one
C:\Games\DLBB-Revival\texture_hook.dll               ← from texture-hook/
C:\Games\DLBB-Revival\EasyAntiCheat\EasyAntiCheat_x64.dll  ← from stubs/
```

To make the LightFX proxy work, you must rename the original
`LightFX.dll` (the real Techland file) to `LightFX_original.dll` BEFORE
dropping in our proxy.

## 7. Get Goldberg Steam Emu

Download from <https://gitlab.com/Mr_Goldberg/goldberg_emulator> or
<https://github.com/Detanup01/gbe_fork>. Use the **experimental** build
(includes `steamclient.dll` + `steamclient64.dll`).

Files to drop in the game folder:
- `steam_api64.dll`
- `steam_api.dll`
- `steamclient64.dll`
- `steamclient.dll`
- `steam_interfaces.txt` (generated by Goldberg's interface-detector tool
  against the **original** Techland `steam_api64.dll`)

The launcher writes `steam_settings/force_account_name.txt`,
`steam_settings/force_steamid.txt`, and
`steam_settings/custom_broadcasts.txt` on login.

## 8. Optional: multi-instance on the same PC

To test with multiple accounts on the same machine, each instance needs:

1. A separate copy of the game folder.
2. A patched mutex in `BadBloodGame.exe`: find the ASCII string
   `ChromeEngine4DIMutex` and change the last character (e.g. to
   `ChromeEngine4DIMute1`, `...e2`, `...e3`). Without this, the game
   refuses to start while another copy is already running.

## 9. Launch

Run the BadBlood-Revival launcher, register/login, and click PLAY. The
launcher writes Goldberg config files based on your account and spawns
`BadBloodGame.exe`.

## Troubleshooting

| Symptom | Likely cause |
|---------|--------------|
| "Start the game using BadBloodGameLauncher.exe" popup | Patch at 0x6F73DE didn't take. Re-run patcher with `--dry-run` to inspect, then for real. |
| "Application load error V:0000065432" | Steam is running. Close it. |
| "Application load error 3:0000065432" | Steamless step (#3) wasn't done. |
| Black screen / immediate exit | Goldberg version mismatch — try the experimental build. |
| "Retrieving data from server" forever | stunnel isn't running on 443, or the cert isn't trusted by the patched engine (but SSL verify should be off — re-check patch 0x6F95AA). |
| "Bad Blood is already running" | Mutex patch (#8) needed for multi-instance. |
| Empty inventory crash at menu | New player without 30 default items. Check DB: `select count(*) from player_items where pls_id = <yours>`. Should be 30+. |
| Shop purchase doesn't persist | Server isn't saving. Check `pls-emu.log` for the POST `/dlbb/shop/itempurchase` line. |
| Can't start a match | Need 6 players. Hardcoded in the game binary. |
